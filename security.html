Executive summary

Solid baseline for a static site. No high-severity findings. Two medium issues — a missing Content-Security-Policy and missing Subresource Integrity on third-party scripts — both worth fixing before this is referenced from a higher-traffic surface.

The remaining low and info findings are typical of a GitHub Pages deployment: server-software disclosure (GitHub Pages signature), no Permissions-Policy header, and a public sitemap. None of them put visitor data at risk; all of them would be cheap wins on a custom-host migration.

Findings · 9 total
Medium Headers / CSP

No Content-Security-Policy header set

Without a CSP, the browser allows scripts and stylesheets from any origin. On a static portfolio the blast radius is low, but defining a strict CSP would prevent injected scripts from loading even if the markup were ever tampered with.

GET / HTTP/2 ↳ no Content-Security-Policy header in response
Medium SRI / external scripts

Third-party scripts loaded without integrity attribute

GSAP, ScrollTrigger, and Lenis are loaded from cdn.jsdelivr.net without integrity/crossorigin. A compromised CDN could substitute malicious code; SRI hashes prevent that.

<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/gsap.min.js" defer></script> ↳ no integrity= attribute · 3 scripts affected
Low Disclosure

Server header reveals hosting platform

The response includes Server: GitHub.com, fingerprinting the hosting platform. Information disclosure on its own isn't dangerous, but it widens the attacker's reconnaissance surface.

HTTP/2 200 OK Server: GitHub.com
Low Headers / Permissions-Policy

Permissions-Policy header missing

No Permissions-Policy means embedded contexts (iframes, third-party scripts) inherit the default-permissive feature set. Setting an explicit policy that disables features you don't use (camera, geolocation, microphone) is good hygiene.

GET / HTTP/2 ↳ no Permissions-Policy header

Scope & authorization

The live scan only ever targets aryanmainkar.github.io — a site I own, where I have explicit authorization to scan. The "try the demo" mode is a pre-rendered example, not a real scan, so paste-any-URL never triggers an outbound request to a third-party site.

That's deliberate. Unauthorized scanning of someone else's infrastructure is legally and ethically off-limits, even with passive checks. If you'd like a real scan of a site you own, get in touch — happy to run it under a written engagement.
